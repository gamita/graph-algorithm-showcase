Fattest Path Algorithm

This demo visualizes a process of Fattest Path Algorithm (Please click Step 1 "Graph" button, then click Step2 "Run" button). This path finding algorithm is not for "Shortest" path, this is for "Fattest" path. To find the fattest path, that is to say, to find a path whose the most narrow edge is the fattest than any other paths. Therefore, this algorithm focuses on the width of edges, not the length of edges. One of motivations to find the fattest path is to find max capacity on the path to the goal node (e.g. logistics, transportation, transmission), even if that path is long path. The number emerging below each node is a maximum width on the way from the start node to its node. For more detail, please check the above wiki link.
